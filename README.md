 Overview
This project demonstrates a simulated **Multi-Factor Authentication (MFA)** rollout, lockout, and user recovery process in a demo identity environment using **Azure AD trial / Google Identity** and **Authenticator apps**.

The goal was to strengthen organizational security posture while ensuring smooth user recovery paths and proper communication protocols.

---

 Scenario
> *Your organization is enabling MFA. You must configure policies, simulate a lockout, and handle user recovery while maintaining security.*

Key tasks:
- Enable and configure MFA policies.
- Simulate user losing access (e.g., phone loss).
- Initiate and document user recovery path.
- Create recovery guide and communication plan.
- Write incident ticket log for lockout and resolution.
